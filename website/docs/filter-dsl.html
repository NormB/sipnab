Filter DSL

sipnab provides a declarative, non-Turing-complete filter language for matching SIP dialogs and their associated RTP streams. Filters can be specified on the CLI with --filter, in the config file under [filter] expression, or typed interactively in the TUI filter dialog (F7).

Grammar

expr = or_expr
or_expr = and_expr ("OR" and_expr)*
and_expr = not_expr ("AND" not_expr)*
not_expr = "NOT" atom | atom
atom = comparison | "(" expr ")"
comparison = field operator value
value = string | number | boolean | regex
string = 'single-quoted' | "double-quoted"
boolean = true | false

Boolean operators (AND, OR, NOT) are case-insensitive. Parentheses nest up to 50 levels deep. Regex patterns are limited to 1 MB to prevent ReDoS.

RTP Quality Fields

For RTP fields, the worst value across all associated streams is used. This matches the typical use case: finding calls with quality problems.

FieldTypeDescription
rtp.mosnumberEstimated MOS score (1.0-5.0). Uses worst stream.
rtp.jitternumberJitter in milliseconds. Uses worst stream.
rtp.lossnumberPacket loss percentage (0.0-100.0). Uses worst stream.
rtp.packetsnumberTotal RTP packets received across all streams.
rtp.orphanedboolHas orphaned RTP streams (no associated SIP dialog).
rtp.codecstringRTP codec name (e.g., PCMU, opus).
rtp.ssrcstringRTP SSRC value (hex string).

Named Aliases

The --problems, --slow-setup, --short-calls, --one-way, and --nat-issues CLI flags are aliases that expand to predefined filter expressions:

AliasExpands To
problems state == 'Failed' OR one_way == true OR rtp.loss > 2.0 OR rtp.jitter > 50.0 OR nat_mismatch == true OR retransmits > 3 OR pdd > 32.0 OR rtp.orphaned == true
slow-setup pdd > 3.0
short-calls duration < 5.0 AND state == 'Completed'
one-way one_way == true
nat-issues nat_mismatch == true

Examples

Basic comparisons

# Calls from a specific user
from.user == '1001'

# Failed calls
state == 'Failed'

# Calls longer than 30 minutes
duration > 1800.0

RTP quality filtering

# Low MOS score
rtp.mos < 3.0

# High jitter
rtp.jitter > 50.0

# Significant packet loss
rtp.loss > 2.0

# Combined quality check
rtp.mos < 3.5 AND rtp.jitter > 30.0

Boolean logic

# Calls from alice OR bob
from.user == 'alice' OR from.user == 'bob'

# Failed INVITE calls (not REGISTER)
method == 'INVITE' AND state == 'Failed'

# Exclude scanners
NOT ua =~ 'friendly-scanner'

# Complex: problem calls from a specific subnet
(rtp.mos < 3.0 OR one_way == true) AND src.ip =~ '^10\.0\.0\.'

Regex matching

# Match any Polycom phone
ua =~ 'PolycomVVX'

# Match SIP scanner User-Agents
ua =~ 'friendly-scanner|sipvicious|sipcli'

# Source IP in a /24 subnet
src.ip =~ '^192\.168\.1\.'

# Call-ID containing a specific domain
call_id =~ '@example\.com$'

CLI usage

# Pass filter on CLI
sipnab --filter "from.user == '1001' AND rtp.mos < 3.0"

# Combine with other flags
sipnab -N --json --filter "state == 'Failed'" -I capture.pcap

# Use in config file
# [filter]
# expression = "method == 'INVITE' AND NOT ua =~ 'scanner'"

Safety Limits

  • Max nesting depth: 50 levels of parentheses
  • Max regex size: 1 MB per regex pattern
  • Evaluation: Non-Turing-complete -- no loops, no recursion, no side effects

These limits prevent ReDoS and stack overflow attacks from malicious filter expressions.
