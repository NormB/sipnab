Installation

From crates.io

The simplest approach. Requires a Rust toolchain (1.80+) and libpcap headers.

cargo install sipnab

To install with all feature flags:

cargo install sipnab --features full

From Source

Prerequisites

  • Rust 1.80+ (nightly recommended for let-chains)
  • libpcap development headers
  • A C compiler (for pcap-sys)

Debian/Ubuntu:

sudo apt install libpcap-dev build-essential

Fedora/RHEL:

sudo dnf install libpcap-devel gcc

macOS:

brew install libpcap
# libpcap is also available from Xcode command-line tools

Build

git clone https://github.com/normbrandinger/sipnab.git
cd sipnab
cargo build --release

The binary will be at target/release/sipnab.

Feature Flags

# Core only (default: capture, SIP, TUI, CLI, JSON)
cargo build --release

# With TLS decryption (pure Rust)
cargo build --release --features tls

# With HEP v2/v3 support
cargo build --release --features hep

# With REST API + Prometheus
cargo build --release --features api

# Everything
cargo build --release --features full

Static Binary (musl)

For a fully static binary with no runtime dependencies:

rustup target add x86_64-unknown-linux-musl
cargo build --release --target x86_64-unknown-linux-musl

The resulting binary is under 5 MB when stripped.

Docker

# Build
docker build -t sipnab .

# Run with host networking (for live capture)
docker run --rm --net=host sipnab -d eth0

# Analyze a pcap file
docker run --rm -v /path/to/files:/data sipnab -N -I /data/capture.pcap

Cross-Compilation

sipnab supports cross-compilation via cross:

cargo install cross
cross build --release --target aarch64-unknown-linux-musl

Permissions

Live packet capture requires either root privileges or the CAP_NET_RAW capability:

# Option A: setcap (recommended)
sudo setcap cap_net_raw=eip target/release/sipnab

# Option B: run as root (sipnab will drop privileges after opening devices)
sudo sipnab -d eth0 --user nobody

sipnab automatically drops privileges after opening capture devices unless --no-priv-drop is specified. See the CLI reference for details.

Verifying the Install

sipnab --version
sipnab --dump-config
