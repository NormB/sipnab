CLI Reference

All command-line flags organized by group. Flags can be combined freely; conflicting flags produce clear error messages at startup.

sipnab [OPTIONS] [BPF_FILTER...]
Capture
FlagTypeDefaultDescription
-d, --device <IFACE> string auto Network interface to capture on. Omit to auto-detect.
-I, --input <FILE> string — Read packets from a pcap/pcapng file instead of live capture.
-O, --output <FILE> string — Write captured packets to a pcap file.
-B, --buffer <MIB> u32 kernel default Kernel capture buffer size in MiB. Increase for high-rate capture to avoid drops.
--snaplen <BYTES> u32 65535 Snapshot length for packet capture in bytes. Reduce to save memory at the cost of truncated payloads.
--portrange <RANGE> string 5060-5061 SIP port range for the auto-generated BPF filter.
--multi-device bool false Capture on all available interfaces simultaneously.
--no-rtp bool false Disable RTP capture and analysis entirely. Reduces CPU and memory usage when you only need SIP.
--bpf-file <FILE> string — Read a BPF filter expression from a file instead of the command line.
-n, --count <N> u64 — Stop after capturing N packets.
--duration <DURATION> string — Stop after capturing for this duration. Accepts 30s, 5m, 1h.
--autostop <CONDITION> string — Autostop condition. filesize:100 (MB), duration:60 (seconds).
--split <CONDITION> string — Split output files. filesize:50 creates 50 MiB chunks.
--replay bool false Replay packets from a pcap file at original timing. Simulates live capture from a recording.
--pcapng bool false Use pcapng format for output files (supports interface metadata, comments).

Example:

sipnab -d eth0 -B 32 --portrange 5060-5080 -O capture.pcap

Positional BPF filter:

sipnab host 10.0.0.1 and port 5060
Mode
FlagTypeDefaultDescription
-N, --no-tui bool false Non-interactive mode (no TUI). Required for batch output flags like --json, --report, --fail2ban.
-c, --calls-only bool false Show only SIP dialogs (calls), hiding standalone messages like OPTIONS pings.
-t, --telephone-event bool false Capture and display RFC 4733 telephone-event (DTMF) RTP payloads.
-q, --quiet bool false Suppress informational output; only show results. Useful when piping to other tools.

Example:

# CLI mode with JSON output
sipnab -N --json -I capture.pcap

# Quiet mode piped to jq
sipnab -N -q --json -I capture.pcap | jq '.call_id'
Matching
FlagTypeDefaultDescription
-i, --ignore-case bool false Case-insensitive matching for header filters and regex patterns.
-v, --invert bool false Invert the match: show messages that do NOT match the filter criteria.
-w, --word bool false Match whole words only (adds word boundary anchors).
--single-line bool false Treat multi-line SIP headers as a single line for matching. Useful for folded headers.
--from <PATTERN> regex — Filter by SIP From header (regex pattern).
--to <PATTERN> regex — Filter by SIP To header (regex pattern).
--contact <PATTERN> regex — Filter by SIP Contact header (regex pattern).
--ua <PATTERN> regex — Filter by User-Agent header. Useful for finding specific phone models or scanners.
--filter <EXPR> string — Advanced filter DSL expression. See Filter DSL reference.

Example:

# Case-insensitive From header match
sipnab --from alice -i

# Invert: show everything except REGISTER traffic
sipnab --filter "method == 'REGISTER'" -v

# Combined: calls from 1001 with poor quality
sipnab --filter "from.user == '1001' AND rtp.mos < 3.0"
Diagnostic Aliases

Shorthand flags that expand to predefined filter DSL expressions. These are convenience wrappers for common troubleshooting patterns.

FlagEquivalent FilterDescription
--problems state == 'Failed' OR one_way == true OR rtp.loss > 2.0 OR rtp.jitter > 50.0 OR nat_mismatch == true OR retransmits > 3 OR pdd > 32.0 OR rtp.orphaned == true Show calls with detected problems: retransmits, timeouts, errors, quality issues.
--slow-setup pdd > 3.0 Show calls with post-dial delay exceeding 3 seconds.
--short-calls duration < 5.0 AND state == 'Completed' Show completed calls shorter than 5 seconds.
--one-way one_way == true Show calls with detected one-way audio (RTP flowing in only one direction).
--nat-issues nat_mismatch == true Show calls where Contact/Via addresses don't match the packet source (NAT traversal issue).

Example:

# Find problem calls in a pcap
sipnab -N --problems -I capture.pcap

# Combine with JSON output for automation
sipnab -N --problems --json -I capture.pcap | jq '.call_id'
Output

Output flags require -N (non-interactive mode) unless --call-report is specified.

FlagTypeDefaultDescription
--json bool false Output results as NDJSON (one JSON object per line).
--json-pretty bool false Output results as pretty-printed JSON (indented, multi-line).
--report bool false Generate a summary report after capture completes.
--call-report <CALL-ID> string — Generate a detailed report for a specific Call-ID. Does not require -N.
--markdown bool false Format report output as Markdown.
--hexdump bool false Include hex dump of SIP payloads.
--delta-time bool false Show delta time between consecutive messages.
-A, --after <N> usize — Show N messages after each match (like grep -A).
--show-empty bool false Show messages with empty bodies (e.g., ACK without SDP).
--line-buffer bool false Flush output after each line. Useful when piping to other tools like tee.
--color <WHEN> string auto Color output mode: auto, always, or never.
--payload-limit <BYTES> usize — Maximum payload bytes to display per message.
-T, --text-dump bool false Dump raw SIP message text (sipgrep -T compatibility).
--wireshark bool false Launch Wireshark with a display filter for the current capture.
--tshark-filter <EXPR> string — Generate a tshark-compatible display filter string.
--fail2ban bool false Output in fail2ban-compatible format for SIP security events.
--group-by <FIELD> string — Group output by field: call-id, from, method, etc.

Example:

# NDJSON streaming, piped to jq
sipnab -N --json -I capture.pcap | jq 'select(.state == "Failed")'

# Detailed call report in Markdown
sipnab --call-report "abc123@192.168.1.1" --markdown -I capture.pcap

# fail2ban integration
sipnab -N --fail2ban --kill-scanner | tee /var/log/sipnab.log
Dialog
FlagTypeDefaultDescription
-l, --limit <N> u64 100000 Maximum number of dialogs to track simultaneously. Increase for high-volume captures.
-R, --rotate bool false Rotate dialog storage when limit is reached. Discards the oldest dialog to make room.
--dialog-track <METHOD> string call-id Dialog tracking method. call-id (standard) or branch (for forking scenarios).
--no-dialog bool false Disable dialog tracking entirely. Message-only mode -- useful for high-throughput SIP proxies where you only care about individual messages.
--tag <TAG> string — Filter dialogs by SIP tag value (From-tag or To-tag).

Example:

# High-volume: track 500K dialogs with rotation
sipnab -l 500000 -R -d eth0
RTP
FlagTypeDefaultDescription
--rtp-interval <SECS> u32 1 RTP statistics reporting interval in seconds.
--max-streams <N> u64 50000 Maximum number of RTP streams to track simultaneously.
--quality-threshold <MOS> f64 3.0 MOS quality threshold for alerts (1.0-5.0 scale). Streams below this trigger quality events.

Example:

# Alert when MOS drops below 3.5
sipnab --quality-threshold 3.5 --on-quality-exec "notify-send 'Low MOS: $SIPNAB_MOS'"
Security
FlagTypeDefaultDescription
--kill-scanner bool false Detect and report SIP scanning activity. Identifies known scanner patterns and User-Agent strings.
--kill-ua <PATTERN> regex — Detect specific User-Agent strings associated with scanners (e.g., friendly-scanner).
--kill-response <CODE> u16 200 SIP response code to use in scanner kill reports. Valid range: 100-699.
--fraud-detect bool false Enable fraud detection heuristics: toll fraud patterns, unusual call patterns, international dialing anomalies.
--reg-flood bool false Detect registration flood attacks (abnormal REGISTER rate from a single source).
--digest-leak bool false Detect digest credential leaks in SIP messages (Authorization headers in plaintext).
--alert <CHANNEL> string (repeatable) — Alert channel. Repeatable: --alert syslog --alert json --alert exec.
--alert-exec <CMD> string — Execute this command when a security alert fires. Receives alert data via environment variables.
--stir-shaken bool false Validate STIR/SHAKEN identity headers on incoming INVITEs.

Example:

# Full security monitoring
sipnab --kill-scanner --fraud-detect --reg-flood --digest-leak \
       --alert syslog --alert json \
       --alert-exec "/usr/local/bin/sip-alert.sh"
Event Execution

Execute external commands when specific events occur. Commands receive event data via environment variables ($SIPNAB_CALL_ID, $SIPNAB_STATE, $SIPNAB_MOS, etc.).

FlagTypeDefaultDescription
--on-dialog-exec <CMD> string — Execute command when a dialog state changes (e.g., Trying, InCall, Completed, Failed).
--on-quality-exec <CMD> string — Execute command when RTP quality drops below --quality-threshold.
--exec-rate-limit <N> u32 10 Maximum exec invocations per second. Prevents fork-bombing on high-traffic systems.

Example:

# Log failed calls to a webhook
sipnab --on-dialog-exec 'if [ "$SIPNAB_STATE" = "Failed" ]; then curl -s -X POST http://hooks.example.com/failed -d "{\"call_id\": \"$SIPNAB_CALL_ID\"}"; fi'
Network Listeners

All network listeners bind to localhost by default. Binding to 0.0.0.0 without TLS produces a warning.

FlagTypeDefaultDescription
--metrics <ADDR> string — Enable Prometheus metrics endpoint. :9090 or 0.0.0.0:9090.
--metrics-auth <TOKEN> string — Bearer token for metrics endpoint authentication.
--api <ADDR> string — Enable REST API endpoint. See REST API reference.
--api-key <KEY> string — API key for REST API authentication. Also reads SIPNAB_API_KEY env var.
--api-tls-cert <FILE> string — TLS certificate for API endpoint.
--api-tls-key <FILE> string — TLS private key for API endpoint.
--api-max-conn <N> u32 100 Maximum concurrent API connections. Returns 503 when exceeded.
-L, --hep-listen <ADDR> string — Listen for HEP (Homer Encapsulation Protocol) packets.
-H, --hep-send <ADDR> string — Send captured packets via HEP to a remote collector (e.g., Homer/SIPCAPTURE).
-E, --hep-parse bool false Parse incoming HEP packets (enable HEP decoding).
--hep-allow <ADDR> string (repeatable) — Allowed source addresses for HEP input. Repeatable.
--hep-rate-limit <N> u64 50000 Maximum HEP packets per second.
--syslog bool false Send security alerts to syslog.

Example:

# Prometheus + REST API with auth
sipnab --metrics :9090 --api :8080 --api-key "s3cret" -d eth0

# Receive HEP from Homer agents
sipnab -L 0.0.0.0:9060 -E --hep-allow 10.0.0.0/24
TLS / Decryption

Requires the tls, tls-wolfssl, or tls-openssl feature flag at build time.

FlagTypeDefaultDescription
-k, --tls-key <FILE> string — TLS private key file for SIP-TLS decryption (RSA key exchange only).
--keylog <FILE> string — TLS key log file (NSS SSLKEYLOGFILE format). Supports TLS 1.2 and 1.3 with any cipher suite.
--keylog-watch bool false Watch key log file for new entries. Enables live decryption as new TLS sessions are established.
--dtls-keylog <FILE> string — DTLS key log file for SRTP key extraction.
--srtp-keys <FILE> string — SRTP master keys file for RTP decryption (SDES key exchange).
--pcap-export-mode <MODE> string decrypted Pcap export mode for encrypted traffic: decrypted (write plaintext) or original (write ciphertext).
--allow-coredump bool false Allow core dumps. By default, sipnab disables core dumps to prevent key material from being written to disk.

Example:

# Decrypt SIP-TLS with keylog file
SSLKEYLOGFILE=/tmp/keys.log sipnab --keylog /tmp/keys.log -d eth0

# Live keylog watching
sipnab --keylog /tmp/keys.log --keylog-watch -d eth0
Privilege
FlagTypeDefaultDescription
--user <USER> string — Drop privileges to this user after opening capture devices.
--no-priv-drop bool false Do not drop privileges after opening capture devices. Use with caution.
--chroot <DIR> string — Chroot to this directory after initialization. Limits filesystem access for defense-in-depth.

Example:

# Production deployment: drop to unprivileged user + chroot
sudo sipnab -d eth0 --user sipnab --chroot /var/lib/sipnab
Resource Limits
FlagTypeDefaultDescription
--max-reassembly <N> u64 10000 Maximum concurrent TCP/TLS reassembly sessions. Limits memory usage under SYN flood or fragmentation attacks.

Additional limits are set via other flags: --limit (dialogs, default 100K), --max-streams (RTP, default 50K), --hep-rate-limit (HEP, default 50K/s), --api-max-conn (API, default 100).

Config
FlagTypeDefaultDescription
-f, --config <FILE> string — Path to configuration file. Must exist if specified.
-F, --no-config bool false Skip loading any configuration file. Use built-in defaults only.
-D, --dump-config bool false Dump the effective configuration as TOML and exit. Useful for debugging config issues.

See the Configuration reference for the TOML file format.
