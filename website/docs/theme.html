Theme

sipnab's TUI uses 11 semantic color slots. Every slot has a sensible default that works on dark terminals. Override any slot in the [theme] section of your config file.

Color Slots

SlotDefaultPreviewUsed For
background reset Terminal background. reset inherits the terminal's own background color.
foreground white Default text color for all views.
header cyan Status bar, column headers, endpoint labels in call flow.
selected yellow Currently selected row, cursor position, focused item.
accent magenta Correlation info, PDD values, extended flow labels.
good green Positive quality (MOS > 4.0), success states (InCall, Registered), 2xx responses.
warning yellow Medium quality (MOS 3.0-4.0), caution states (Ringing), 3xx/CANCEL responses.
bad red Poor quality (MOS < 3.0), failure states (Failed), 4xx/5xx/6xx responses, scanner alerts.
muted dark_gray Separators, pipe characters, disabled text, timestamps, less important info.
border white Widget borders, panel frames, table lines.
highlight — — Legacy alias for selected. Kept for backward compatibility. If both are set, selected wins.

The TUI also uses a hardcoded status_bg color (#303040, dark blue-gray) for the status bar background. This is not configurable but is chosen to be readable on both dark and light terminals.

How Colors Are Applied

sipnab maps semantic color slots to specific TUI elements:

  • Call list: foreground for text, selected for cursor row, header for column headers, dialog state uses good/warning/bad by state
  • Call flow: header for endpoint labels, arrow colors cycle through a palette based on color mode (method/call-id/cseq), accent for PDD and correlation labels, muted for timestamps
  • RTP streams: good/warning/bad for MOS quality indicators, accent for orphaned stream markers
  • Status bar: header text on dark blue-gray background
  • Borders: border for all panel frames
