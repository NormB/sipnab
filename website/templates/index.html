Capabilities

Everything, in one binary

See Everything

Interactive TUI with multi-participant ladder diagrams, auth collapse, retransmit folding, mark+delta timing. Four timestamp modes, message diff, split view with resizable panels.

Analyze Everything

First-class RTP: jitter, loss, MOS per stream. RTCP XR. Comfort noise detection. One-way audio diagnosis. NAT mismatch detection. SDP offer/answer timeline. Post-dial delay and setup time measurement.

PCAP, PCAP-NG, TXT, Mermaid sequence diagrams, JSON, NDJSON, Markdown call reports, fail2ban format, Wireshark filters, tshark display filters. Pipe NDJSON to jq for custom pipelines. Tick sngrep-style checkboxes to save just the dialogs you select, or export everything at once.

Quick Start

Up and running in one command

bash 
# Build from source
git clone https://github.com/NormB/sipnab.git
cd sipnab && cargo build --release

# Capture live SIP traffic
sudo sipnab -d eth0

# Analyze a pcap file
sipnab -I capture.pcap

# CLI mode with JSON output
sipnab -N --json -I capture.pcap

# Show only problematic calls
sudo sipnab -N -d eth0 --problems

# Filter specific calls
sipnab --filter "from.user == '1001' AND rtp.mos < 3.0"

# Detailed call report
sipnab -I capture.pcap --call-report "abc123@host" --markdown

Standards-based

Quality Metrics

sipnab reports the metrics VoIP engineers already work in — grounded in the relevant ITU-T and IETF standards. Every figure is computed from the packets, never guessed.

ITU-T G.107

MOS

Estimated Mean Opinion Score (1–5). The R-factor is mapped to MOS using the G.107 Annex B formula.

ITU-T G.107

R-factor

E-model transmission rating (0–100), estimated from codec, jitter, and packet-loss impairments.

RFC 3550

Jitter

RTP interarrival jitter, tracked per stream as defined by the RTP specification.

RFC 3550

Packet Loss

Lost vs. expected RTP packets, derived from sequence-number gaps per stream.

RFC 3611

Burst / Gap Loss

RFC 3611-style burst/gap analysis separates bursty loss from tolerable random loss.

RFC 3611

RTCP XR

Parses RTCP Extended Reports (PT 207), including the VoIP Metrics report block.
